  What Worked Well

  Why I Needed the Temporary Scripts

  The skill provided Python code as templates/documentation rather than executable artifacts. The issue:

  1. Windows + conda + inline Python = pain - Running multi-line Python via conda run -n bandicoot python -c "..." on
  Windows has quoting nightmares (PowerShell escaping, nested quotes, etc.)
  2. The skill assumed I'd figure out execution - The code blocks were presented as "here's what to run" but not
  packaged in a runnable way

  Suggestions to Improve

  Option A: Include the scripts as files in your plugin
  bandicoot-plugin/
    skills/
      validate.md
    scripts/
      validate_env.py
      validate_data.py

  Then the skill can reference: "Run conda run -n bandicoot python {plugin_path}/scripts/validate_env.py"

  Option B: Simpler, single-purpose commands

  Instead of one big validation script, break into small inline-safe commands:
  conda run -n bandicoot python -c "import bandicoot; print(bandicoot.__version__)"

  Option C: Have the skill instruct me to create the script

  Explicitly say: "Create a file called validate.py with the following content, then run it."

  The Core Issue

  The skill told me what to validate but left how to execute it ambiguous. For cross-platform reliability, either:
  - Ship executable scripts with the plugin
  - Use very simple one-liner commands
  - Or explicitly instruct to create temp files (which is what I ended up doing anyway)

  # Bandicoot: Validate

  Validate your Bandicoot environment and data files.

  ## Usage

  conda run -n bandicoot python {scripts}/validate.py [options]

  ## Options
  - `--env-only` - Only check environment, skip data validation
  - `--user USER_ID` - User ID to validate
  - `--data PATH` - Path to data directory
  - `--antennas PATH` - Path to antennas file

  ## Examples

  # Environment only
  conda run -n bandicoot python {scripts}/validate.py --env-only

  conda run -n bandicoot python {scripts}/validate.py --user sample_user_PRIMARY --data data/
